Page-cache acceptance is not the persistence contract. Only one incomplete final record may be treated as torn. Remote storage adds protocol, cache, server, and failover semantics that must be named.

Compact mental model

Authenticate the peer if required; still distrust every byte it is authorized to send.

Validate each boundary before pointer arithmetic, allocation, copy, or shift. TLS normally protects application bytes above TCP and can authenticate peers through PKI; IPSec protects IP-layer traffic between configured endpoints. Neither supplies application authorization, resource budgets, safe parsers, or replay/idempotency policy.

Primary source. Read the official LLVM libFuzzer documentation, especially corpus, sanitizer, and bounded-run controls. Use the local networking guide for the TLS/PKI/IPSec boundary map.

Core lane

  1. In capstone/work/src/protocol.cpp, reject an empty command, embedded NUL, unsafe/over-128-byte keys, missing/extra arguments, and wire lengths over max_payload before allocation.
  2. In capstone/work/src/packet.cpp, prove every Ethernet/VLAN/IP/TCP offset and declared header length fits before reading it. Invalid packets return a structured invalid result, not a partial success.
  3. In capstone/work/src/server.cpp, retain finite receive/send timeouts and bounded admission. Decide whether malformed, oversized, slow, and overloaded peers receive an error or immediate close; avoid reflection/amplification.
  4. Run ./scripts/check.sh work day12, then the ASan/UBSan build from Day 10.
  5. Create capstone/work/experiments/day12_fuzz.cpp with LLVMFuzzerTestOneInput. Construct a string_view over the supplied bytes and call parse_command; also pass the same bytes to decode_ethernet_ipv4_tcp. It must contain no assertions about random input and no unbounded logging.
