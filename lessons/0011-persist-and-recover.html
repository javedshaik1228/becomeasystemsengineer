Compact mental model

Write intent durably, then mutate memory, then acknowledge; replay repeats only complete valid intent.
  1. Frame a deterministic mutation record.
  2. Append every byte or fail.
  3. Synchronize to the promised durability boundary.
  4. Apply memory state and acknowledge.
  5. On restart, replay complete records in order.
EventRecovery ruleClient-visible promise
Crash before complete recordIgnore one torn final recordMutation was not acknowledged
Crash after durability syncReplay the complete recordAcknowledged mutation returns
Malformed complete recordStop and report corruptionNever silently invent state

write can stop at the page cache. fsync asks the local filesystem/device stack to persist the file's data and required metadata, subject to its contract. Creation and rename can require syncing the containing directory. NFS stable writes/COMMIT and SMB durable handles, leases, caching, and failover introduce different client/server promises; learn the semantics and diagnostic questions, not imaginary two-week mastery.

Primary source. Read return values, errors, and directory caveats in Linux fsync(2). Use the local storage and reliability guide for the bounded NFS/SMB comparison.
