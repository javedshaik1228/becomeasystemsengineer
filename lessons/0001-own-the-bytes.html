The model: stream → header → bounded payload

TCP provides a reliable, ordered byte stream, not records. RFC 9293 states this directly. Your protocol therefore owns message boundaries: read exactly four bytes, decode an unsigned big-endian length, reject it if it exceeds policy, then read exactly that many payload bytes. Short reads are progress, EOF is a state, and errors retain errno.

  1. Accumulate header
  2. Decode length
  3. Validate bound
  4. Allocate payload
  5. Accumulate bytes
  6. Return frame

Use std::byte for uninterpreted storage, std::span for a non-owning bounded view, and fixed-width unsigned integers for wire fields. Review the lifetime sheet and selected TCP concepts in RFC 9293 §2.2.
