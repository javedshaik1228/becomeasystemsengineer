Core lane

  1. Create capstone/work/evidence/day-10/, then run ./scripts/check.sh work day10. In capstone/work/src/protocol.cpp, ensure every failed syscall captures errno before any later library call can overwrite it.
  2. Debug the test's intentional invalid descriptor. Break on netforge::read_exact, run Day 10, step through the failed read, inspect errno, list threads, and save a core:
mkdir -p capstone/work/evidence/day-10
gdb --args build/work/netforge-tests day10
(gdb) set logging file capstone/work/evidence/day-10/debugger.txt
(gdb) set logging enabled on
(gdb) break netforge::read_exact
(gdb) run
(gdb) next
(gdb) print errno
(gdb) info threads
(gdb) bt
(gdb) generate-core-file capstone/work/evidence/day-10/day10.core

macOS LLDB: breakpoint set -n netforge::read_exact, run, thread backtrace all, and process save-core ....

  1. Configure separate sanitizer trees; never combine ASan and TSan:
cmake -S capstone -B build/work-asan -DNETFORGE_TRACK=work \

ctest --test-dir build/work-asan --output-on-failure \
  2>&1 | tee capstone/work/evidence/day-10/asan.txt

If you get stuck
