This is a baseline-only loopback workload over blocking persistent sessions with one request outstanding per session; it is not production server throughput. The C++ driver performs warmup and reports p50/p95/p99/max for measured request round trips without spawning a process per request. Its wall-rate still includes connection and warmup time. Record run-to-run variance and one hypothesis before tuning. Never discard a slow run without a stated environmental reason.
