Recall before notes 10 min

  1. Why does readiness not promise a complete frame?
  2. What policy stops pending output from growing forever?
  3. What does EAGAIN mean for retained parse state?
Check only after answering aloud

Readiness permits one non-blocking attempt. High/low-water marks and rejection bound work. EAGAIN keeps the partial cursor and returns control to the event mechanism.

Compact mental model

An IPC mechanism moves bytes; an ownership protocol makes process failure survivable.
MechanismNatural contractHidden cost
PipeOne-way byte stream, simple parent/child flowTwo pipes for duplex; inherited writers delay EOF
Unix socketDuplex framed control channel; local peer identityStill a stream; framing and backpressure remain
Shared memoryShared bytes with minimal copyingNo message, synchronization, ownership, or recovery semantics

After fork, both processes inherit both socketpair endpoints. EOF appears only after every writer reference closes. Shared memory replaces copying with coordination: atomics or process-shared synchronization, publication order, crash cleanup, and versioning become your job.

Primary source. Read the creation, descriptor, and protocol contract in POSIX socketpair(). Use the local IPC comparison for pipes and shared memory.

Core lane

  1. Run ./scripts/check.sh work day09; this proves the framing API works over an AF_UNIX socketpair.
  2. Create capstone/work/experiments/day09_ipc.cpp. Call socketpair(AF_UNIX, SOCK_STREAM, 0, pair), then fork().
  3. Immediately after the fork, parent closes/releases the child endpoint; child closes/releases the parent endpoint. Wrap each retained descriptor in UniqueFd.
  4. Parent sends control with send_frame. Child receives it, replies ACK control, and exits with _exit. Parent receives, calls waitpid, and reports both protocol and exit status.
  5. Add a --break-tail mode: child writes a valid length plus only part of its body, then exits. Parent must report EOF/truncation and must not hang.
