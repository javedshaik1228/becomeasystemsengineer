Recall before reading

Define a data race without saying “two threads at once.” What shared state does a mutex protect? Why can a condition-variable wait return when no useful work exists? What must be true before notifying?

The model: invariant, predicate, transition

A mutex does not protect code; it protects a stated relationship among data. For the queue: size never exceeds capacity; after close no push succeeds; pop drains existing items, then returns empty. Producers wait for closed || size < capacity. Consumers wait for closed || !empty. Every state transition that can satisfy another thread's predicate happens while holding the same mutex, followed by notification.
