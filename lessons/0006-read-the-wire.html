Draw Ethernet, optional 802.1Q tag, IPv4, and TCP in order. Which fields determine the next header offset? What is network byte order? Why must IP total length agree with captured bytes?

The model: validate the envelope before its contents

Decode fixed-width fields byte by byte or with safe copies, then convert endianness. Never map hostile bytes onto a C++ struct and hope alignment, padding, lifetime, and host endianness match. Read the TCP header in RFC 9293 §3.1 and use the networking reference.
