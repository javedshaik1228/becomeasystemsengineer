Which write releases an order to the consumer? What does a failed try_push mean to the caller?

Mental model

Producers may validate and enqueue, but only the consumer mutates books. Acceptance, queueing, and matching are distinct states. Shutdown must stop admission, drain accepted commands, then join.

Build

  1. Create producer threads that call only try_push.
  2. Create exactly one engine consumer.
  3. Choose and document retry, reject, or load-shed behavior when full.
  4. Implement stop-admission and drain-before-exit semantics.
  5. Compare concurrent pipeline fills with direct sequential replay.

Break it deliberately

Use a tiny queue, slow the consumer, request shutdown while full, and repeat the same seeded replay. No accepted order may disappear.
