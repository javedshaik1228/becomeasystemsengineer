Recall before reading

What state belongs to one book? What state belongs across all books? Why is deterministic event order important for replay?

Mental model

Many threads may eventually submit commands, but exactly one matching consumer mutates books. Prepare symbols before traffic so allocation and map growth do not surprise the hot path.

Build

  1. Implement prepare_symbol without replacing existing books.
  2. Reject unprepared symbols instead of silently allocating during submit.
  3. Assign monotonic sequences when input sequence is zero.
  4. Route each order to exactly one symbol book.
  5. Add symbol-aware cancellation if your interface supports it.

Break it deliberately

Submit two symbols at the same price, an unknown symbol, and replayed orders with existing sequences. Confirm no cross-symbol fill can occur.

cmake -S capstone/matching/work -B build/matching-work -DMATCHING_STEP=5
cmake --build build/matching-work -j
ctest --test-dir build/matching-work --output-on-failure
