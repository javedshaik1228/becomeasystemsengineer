Cold retrieval

Explain the full order path, every ownership transfer, price-time priority, two backpressure points, and one Redis/Kafka failure without looking at code.

Mental model

Throughput answers “how much”; tail latency answers “how bad for the slowest requests.” Benchmark generation stays outside the timed region. Run ASan/UBSan separately from TSan. CMake’s official CTest tutorial explains the repeatable test harness.

Build

  1. Add separate direct-core and queued-pipeline benchmarks.
  2. Report orders, fills, elapsed time, throughput, p50, p95, and p99.
  3. Warm up, pin assumptions, and record compiler/build/hardware.
  4. Run correctness, ASan/UBSan, and TSan configurations.
  5. Profile one run and record the dominant function.
  6. Write work/evidence/final/ARCHITECTURE.md and a failure matrix.

Break your own claim

Move mock generation into the timed region, enable broker calls on the engine thread, and use a tiny queue. Observe how each changes the metric and explain why the original measurement boundary matters.
