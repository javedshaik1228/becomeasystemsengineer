Mental model

A seed is only useful when the algorithm, schema, initial state, and ordering are also fixed. Parsing is an adapter; it must not leak into the order book.

Build

  1. Implement a deterministic generator in generate_mock_orders.
  2. Keep symbols, prices, and quantities within documented bounds.
  3. Add CSV read/write helpers and include every field.
  4. Add a small generator CLI and replay CLI.
  5. Compare the fill output of two runs byte-for-byte.

Break it deliberately

Try malformed rows, invalid enum letters, numeric overflow, empty files, and a different seed. Report row numbers on parse errors.
