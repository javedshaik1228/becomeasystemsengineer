POSIX I/O and sockets

Treat every syscall as a partial state transition: inspect its return value, preserve framing, bound buffers, and make close semantics explicit.

The descriptor model

A file descriptor (FD) is a process-local integer referring through the process table to an open file description or kernel object. Duplicated descriptors can share file offset and status flags. Descriptor ownership still needs RAII; the integer itself is not the resource. Start with socket(2) and the ownership field guide.

TCP server lifecycle

  1. Resolve.
    getaddrinfo; support IPv4/IPv6 candidates.
  2. Create/configure.
    socket, CLOEXEC, options, nonblocking if chosen.
  3. Bind/listen.
    Address plus port; backlog is not total capacity.
  4. Accept/process.
    Loop on accepted sockets; frame the byte stream.
  5. Drain/close.
    Stop accepts, finish policy, shutdown, release owner.

Linux documents socket-layer behavior in socket(7) and TCP-specific behavior in tcp(7). Keep portable POSIX semantics separate from Linux-only flags and APIs.

TCP is a byte stream; your protocol supplies messages

// Fixed 4-byte big-endian length prefix; sketch, not complete production code.
enum class Parse { need_more, frame_ready, protocol_error };

Parse next_frame(std::span<const std::byte> input, Frame& out) {
  if (input.size() < 4) return Parse::need_more;
  const std::uint32_t n = decode_be32(input.first<4>());
  if (n > kMaxFrame) return Parse::protocol_error;
  if (input.size() < 4u + n) return Parse::need_more;
  out.assign(input.subspan(4, n));
  return Parse::frame_ready;
}

One send need not match one recv. Reads can split a frame or coalesce many frames. A correct parser preserves incomplete bytes, validates lengths before allocation/arithmetic, emits all complete frames, and caps buffered data. A correct writer preserves unsent suffixes after a short write.

Blocking write-all loop

std::size_t sent = 0;
while (sent < bytes.size()) {
  const auto n = ::send(fd, bytes.data() + sent,
                        bytes.size() - sent, MSG_NOSIGNAL);
  if (n > 0) { sent += static_cast<std::size_t>(n); continue; }
  if (n < 0 && errno == EINTR) continue;
  throw SocketError(errno); // include operation and peer context
}

MSG_NOSIGNAL is available on Linux; macOS commonly uses SO_NOSIGPIPE. Verify platform APIs. A write loop must also integrate deadlines/cancellation for a real service.

Readiness is permission to try, not completion

Nonblocking mode converts potential waiting into EAGAIN. A readiness API says an operation is likely to make progress; by the time you call it, state can have changed. Drain until EAGAIN when using edge-triggered modes, and retain per-connection input/output state.

ConcernPolicy to stateObservable signal
Slow readerBound per-connection output; pause reads, shed, or close at thresholdQueued output bytes and oldest age
Slow senderIdle/read deadline and maximum partial-frame bytesPartial input bytes and last progress
Accept overloadConnection cap, admission policy, FD reserveActive connections, accepts rejected, EMFILE
Worker overloadBound job queue and reject/degrade behaviorQueue depth, wait time, rejected jobs

Linux epoll(7) and macOS kqueue/kevent differ in event registration and reporting. The core interview invariant is portable: nonblocking state machines retain partial progress and never allow queues to grow without policy.

Signals and orderly shutdown

Most library and C++ operations are not async-signal-safe. Keep a signal handler minimal: set a lock-free flag or write a byte to a self-pipe/event mechanism, then let the normal event loop perform shutdown. Consult signal-safety(7). A defensible sequence is: stop admission, request cancellation, wake waiters, drain or reject by policy, call shutdown where wire semantics matter, join workers, then release descriptors.
