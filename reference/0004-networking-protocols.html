Networking protocols

Follow a packet one boundary at a time: identify the layer, validate lengths, explain the state, and predict the next observable event.

Safe packet parsing

  1. Bound.
    Check captured bytes for the smallest header.
  2. Decode.
    Read fields without unaligned or aliasing-unsafe casts.
  3. Validate.
    Version, header length, total length, options, next protocol.
  4. Advance.
    Use checked offsets and lengths; stop on truncation.
std::optional<std::uint16_t> read_be16(
    std::span<const std::byte> bytes, std::size_t offset) {
  if (offset > bytes.size() || bytes.size() - offset < 2) return std::nullopt;
  const auto hi = std::to_integer<std::uint16_t>(bytes[offset]);
  const auto lo = std::to_integer<std::uint16_t>(bytes[offset + 1]);
  return static_cast<std::uint16_t>((hi << 8) | lo);
}

Network byte order is big-endian. Avoid overlaying a C++ struct on packet bytes: padding, alignment, object lifetime, aliasing, and host byte order make it brittle. Validate arithmetic as available - offset >= needed after proving offset <= available, avoiding overflow in offset + needed.

TCP reasoning sheet

The current core TCP specification is RFC 9293. TCP establishes synchronized sequence spaces, acknowledges bytes, retransmits when delivery evidence is missing, preserves order to the application, and applies both receiver flow control and network congestion control.

ConceptWhat to sayWhat not to imply
Three-way handshakeSYN, SYN-ACK, ACK synchronize initial sequence numbers and establish state in both endpointsIt does not authenticate the peer or negotiate application framing
Sequence / ACKSequence numbers identify byte positions; cumulative ACK commonly identifies the next expected byteOne ACK is not one application message
Flow controlReceiver window prevents overrunning the receiver's advertised capacityIt is not congestion control
Congestion controlSender adapts in-flight data to inferred network capacityIt cannot fix application-level unbounded queues
FIN / half-closeOne direction can reach EOF while the other still sendsclose() is not an application-level delivery acknowledgement
RSTAbortive reset invalidates the connection stateIt does not identify the root cause by itself
TIME_WAITThe active closer retains state to handle delayed segments and reliable closeIt is not automatically a leak or OS bug

UDP and application responsibility

UDP preserves datagram boundaries and supplies ports plus a checksum, but it does not promise delivery, ordering, duplicate suppression, congestion control, or a connection handshake. If the application needs these, its protocol must define them. "UDP is faster" is workload- and design-dependent; reliability logic can dominate.

CIDR and routing essentials

A prefix such as 10.20.4.0/24 states that the first 24 bits identify the network prefix. A host uses its routes to choose a next hop; routers use longest-prefix match among applicable routes. ARP resolves an IPv4 next-hop address on the local link - it does not cross routers. IPv6 uses Neighbor Discovery rather than ARP.

QuestionReasoning move
Is destination on-link?Apply the interface prefix/mask; otherwise choose a gateway from the route table
Which route wins?Longest matching prefix, then platform policy/metric among comparable routes
Why can ping work but TCP fail?Different protocol/ports, ACL/firewall state, PMTU, service/listener, NAT, or asymmetric path
MTU vs MSS?MTU bounds an L2/L3 packet on a path/link; TCP MSS advertises maximum TCP payload segment size

Interview traps

TrapWhy it failsStronger answer
"ARP finds any Internet host's MAC."ARP operates on the local link; a host normally resolves the next-hop router.Separate destination IP from next-hop L2 address.
"TCP preserves messages."TCP exposes an ordered byte stream.Define length-prefix, delimiter, or fixed-size application framing.
"A checksum makes data secure."Error detection is not cryptographic authentication or confidentiality.State threat model and use authenticated cryptography/TLS.
"UDP is always faster."Workload, loss, user-space reliability, kernel path, and latency objective matter.Compare measured end-to-end requirements and failure semantics.
"TIME_WAIT means leaked sockets."It is a normal TCP state protecting connection semantics.Identify active closer, rate, tuple reuse, and actual resource pressure.
Confuse MTU and MSSThey apply at different layers and account for different headers.Draw link MTU, IP packet, TCP header, and payload.
"TLS replaces TCP."TLS normally runs over a reliable transport and protects application bytes.Place each protocol in the path and state its contract.
Cast packet bytes to a header structAlignment, padding, lifetime, aliasing, and byte order can break the parse.Bounds-check and decode fields explicitly.
